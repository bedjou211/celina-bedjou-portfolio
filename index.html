🚆 Système de Vigilance Numérique – Hackathon SNCF (1ère place)

Problématique : Les incidents ferroviaires sont souvent détectés tardivement, ce qui impacte fortement les usagers et les opérations.

Solution développée :

  • Conception d’une architecture multi-agents avec 7 agents spécialisés :
    • Collecte des données temps réel (APIs GTFS-RT, SIRI Lite)
    • Nettoyage et normalisation des flux
    • Analyse des anomalies
    • Prise de décision
    • Génération de rapports via LLM
  • Détection d’anomalies sur séries temporelles combinant :
    • Z-score contextualisé basé sur historique
    • Analyse de dérive (slope)
    • Persistance temporelle (réduction des faux positifs)
  • Conception d’un algorithme de scoring intégrant :
    • Gravité opérationnelle
    • Impact utilisateur (heures de pointe, flux)
    • Confiance statistique
  • Utilisation d’un LLM pour :
    • Générer des synthèses d’incidents
    • Estimer les temps de reprise du trafic
  • Pipeline data :
    • Ingestion → Snowflake
    • Traitement → Python
    • Monitoring → Streamlit

Résultat : Détection des incidents plusieurs minutes avant les systèmes traditionnels.
