🚆 Système de Vigilance Numérique (SNCF Hackathon 🥇)

🎯 Problème métier :
Les incidents ferroviaires sont détectés tardivement à partir de signalements humains ou systèmes passifs, entraînant des retards et une mauvaise expérience utilisateur.

🧠 Objectif :
Construire un système capable de détecter automatiquement les anomalies en temps réel et d’anticiper les perturbations avant leur confirmation officielle.

⚙️ Architecture technique :

  • Conception d’une architecture multi-agents (7 agents IA) :
    • Ingestion des flux temps réel (GTFS-RT, SIRI Lite APIs)
    • Nettoyage et normalisation des données
    • Détection d’anomalies
    • Scoring et priorisation
    • Génération de rapports explicatifs via LLM
  • Modélisation des séries temporelles :
    • Z-score dynamique basé sur historique
    • Détection de dérive (trend analysis)
    • Filtrage de bruit via persistance temporelle
  • Système de décision :
    • Score = Gravité × Impact utilisateur × Confiance statistique
    • Priorisation des incidents critiques
  • Pipeline data engineering :
    • Ingestion → Snowflake
    • Processing → Python (Pandas, NumPy)
    • Visualisation → Streamlit dashboard

📊 Résultat :
Détection des anomalies plusieurs minutes avant les systèmes existants, avec réduction des faux positifs grâce au scoring multi-critères.

Python Multi-Agent Time Series Snowflake LLM Streamlit

🤖 AI Customer Support System (RAG + NLP)

🎯 Problème métier :
Les systèmes de support client traditionnels sont non contextualisés, lents et incapables de prioriser les tickets intelligemment.

🧠 Objectif :
Créer un système capable de comprendre, prioriser et répondre automatiquement aux demandes clients.

⚙️ Architecture :

  • Pipeline RAG (Retrieval Augmented Generation)
  • Indexation vectorielle des documents métier
  • Recherche sémantique via embeddings
  • Analyse de sentiment pour priorisation automatique
  • Génération de réponses via LLM contextualisé

🔧 Détails techniques :

  • Embedding des tickets et base de connaissances
  • Similarité cosinus pour retrieval
  • Classification des requêtes (urgence, type, intention)
  • Génération de réponse conditionnée par contexte

📊 Résultat :
Amélioration de la qualité des réponses et réduction du temps de traitement des tickets.

🧬 PRISME NLP (INSERM)

🎯 Problème :
Les chercheurs ont besoin d’outils pour estimer l’impact potentiel des publications scientifiques avant leur diffusion.

🧠 Objectif :
Construire un modèle NLP capable de représenter sémantiquement des publications scientifiques et prédire leur impact.

⚙️ Pipeline technique :

  • Prétraitement de corpus scientifiques (cleaning, tokenization)
  • Utilisation de Transformers spécialisés :
    • SciBERT (domaine biomédical)
    • SPECTER2 (représentation de publications)
  • Création d’embeddings sémantiques
  • Feature engineering pour variables bibliométriques
  • Modèle supervisé de prédiction d’impact

📊 Résultat :
Amélioration de la compréhension des relations entre publications et aide à la priorisation scientifique.

🧠 NutriRAG – Analyse nutritionnelle intelligente

🎯 Problème :
Les données nutritionnelles sont complexes, non structurées et difficiles à exploiter pour la segmentation.

🧠 Objectif :
Transformer des données nutritionnelles brutes en insights exploitables via ML non supervisé.

⚙️ Méthodologie :

  • Feature engineering nutritionnel
  • Réduction de dimension :
    • PCA (structure linéaire)
    • UMAP (structure non linéaire)
  • Clustering :
    • K-Means
    • DBSCAN
    • Clustering hiérarchique

📊 Résultat :
Segmentation cohérente des aliments et extraction de patterns nutritionnels exploitables.
